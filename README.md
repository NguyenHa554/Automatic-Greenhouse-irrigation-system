# Irrigation Model

This project analyzes greenhouse data and builds a model to predict irrigation volume over 3-hour cycles. The current pipeline uses greenhouse climate data, outdoor weather data, and slab sensor data to engineer features, train a LightGBM regression model, and optionally export the trained model as a native library with Treelite.

## Project Contents

- `Analysis.ipynb`: main notebook for data preprocessing, feature engineering, model training, and evaluation.
- `GreenhouseClimate.csv`: greenhouse climate data, including temperature, humidity, CO2, PAR, irrigation information, and control signals.
- `Weather.csv`: outdoor weather data, including radiation, rain, humidity, temperature, and wind.
- `GrodanSens.csv`: slab sensor data, including EC, water content, and slab temperature.
- `TomQuality.csv`: tomato quality measurements at several time points.
- `processed_greenhouse_data_clean.csv`: cleaned and feature-engineered dataset resampled to 3-hour cycles.
- `lgbm_irrigation_model.joblib`: saved LightGBM model.
- `s.joblib`: additional joblib model or artifact.
- `import joblib.py`: script that converts the saved LightGBM model into C/native library output using Treelite.
- `irrigation_model/`: output directory generated by the Treelite export step.

## Environment Requirements

The project uses Python with common data analysis and machine learning libraries:

```bash
pip install pandas numpy matplotlib seaborn scikit-learn imbalanced-learn lightgbm joblib treelite treelite-runtime
```

If you run the Treelite export step, your machine also needs a compatible C/C++ compiler toolchain, such as GCC.

## Data Processing Pipeline

`Analysis.ipynb` performs the following main steps:

1. Load `GreenhouseClimate.csv`, `Weather.csv`, and `GrodanSens.csv`.
2. Merge the datasets on the `%time` column.
3. Convert `%time` from Excel serial-day format to `datetime`.
4. Keep the main environmental features, such as `Tair`, `Rhair`, `CO2air`, `HumDef`, `Tot_PAR`, `Iglob`, `Rain`, `EC_slab1`, and `WC_slab1`.
5. Create additional features:
   - `VPD` from air temperature and relative humidity.
   - Time-based features such as `hour`, `dayofweek`, and `PartOfDay`.
   - 30-minute rolling features such as `Tair_30m_avg` and `Iglob_30m_sum`.
   - Lag/difference features such as `WC_slab1_30m_ago` and `Tair_30m_diff`.
   - Interaction features such as `PAR_x_HumDef` and `Tair_x_Iglob`.
6. Calculate `Watering_Amount` from the positive difference of `Cum_irr`.
7. Resample the data into 3-hour cycles.
8. Save the processed dataset to `processed_greenhouse_data_clean.csv`.

## Model Training

The prediction target is:

```text
Watering_Amount
```

The notebook reads `processed_greenhouse_data_clean.csv`, splits the data into train and test sets in chronological order with `shuffle=False`, then trains a `lightgbm.LGBMRegressor`.

The notebook evaluates the model with:

- `RMSE`
- `MAE`
- `R2 Score`
- `Recall` and `Precision` as a classification proxy with an irrigation threshold of `0.1`

## How to Run

Open and run the notebook:

```bash
jupyter notebook Analysis.ipynb
```

Or:

```bash
jupyter lab Analysis.ipynb
```

After running the preprocessing cells, `processed_greenhouse_data_clean.csv` will be regenerated. After running the training section, you can save the trained LightGBM model as a `.joblib` file if needed.

## Exporting the Model with Treelite

The `import joblib.py` script loads `lgbm_irrigation_model.joblib`, converts the LightGBM model with Treelite, and compiles it into a native library:

```bash
python "import joblib.py"
```

The default output is written to:

```text
irrigation_model/
```

The script currently configures the native library output path as:

```text
irrigation_model/irrigation_model.so
```

On Windows, depending on the compiler toolchain and Treelite configuration, you may need to adjust the output extension or compiler settings.

## Data Notes

- The source CSV files use `%time` in Excel serial-day format and should be converted with the `1899-12-30` origin.
- Some columns in the source data contain mixed data types, so the notebook uses `low_memory=False` and converts values with `pd.to_numeric(..., errors='coerce')`.
- The processed dataset is resampled to 3-hour cycles and uses `datetime` as the index.
